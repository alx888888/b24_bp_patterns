# Примеры JSONPath

- Раздел: Примеры использования
- URL: https://it-solution.kdb24.ru/public/restactivity/a254147/
- Последнее изменение: 27.03.2025 09:36:28

Будем разбирать работу JSONPath на информации, которую мы получим   
с помощью REST-метода user.get. При вызове данного метода мы можем   
наблюдать следующее (см. скрин ниже).   

Нам пришёл ответ, в котором содержится много объектов внутри списка,   
также внутри каждого объекта есть поля с данными.

Задача, которую решает JSONPath: помогает достать необходимые вам   
данные с помощью определенного синтаксиса. 

![](https://it-solution.kdb24.ru/api/v1/froala/file_proxy/370119/3165/739101/?sig=3c6fb3aa543df154454d02020be4546c)

## Получить один объект из списка

Попробуем получить первый объект в списке, для этого введем в JSONPath   
следующую конструкцию $[0] (см. скрин ниже). Чтобы получить последний объект  
 списка в скобках нужно указать $[-1] или ID последнего объекта.   

Таким образом, как указано выше, вы можете получить любой объект списка.

![](https://it-solution.kdb24.ru/api/v1/froala/file_proxy/370123/3165/739109/?sig=dd9066a5ae7c87714020dc136cb5b7d1)

## Получить значение поля объекта

Попробуем у найденного объекта получить значение поля EMAIL,   
для этого вводим в JSONPath $[0].EMAIL (см. скрин ниже).   

Таким образом, вы можете получить значение любого поля в данном   
объекте, просто поменяв значение EMAIL на любое другое, которое вам необходимо.

![](https://it-solution.kdb24.ru/api/v1/froala/file_proxy/370126/3165/739119/?sig=1281ac200a2384c82c58118fb21aa924)

## Получить элемент списка внутри объекта

Теперь сделаем чуть более сложный запрос и получим первое значение   
у списка в поле UF_DEPARTMENT. Для этого вводим в поле $[0].UF_DEPARTMENT[0] (см. скрин ниже). 

Изменяя индекс, вы можете получить любой элемент списка.

![](https://it-solution.kdb24.ru/api/v1/froala/file_proxy/370127/3165/739123/?sig=6c9541ccd98bd7fead36d6ca1fddb601)

## Получить по каждому объекту только значения определенного поля

Вернемся к первоначальному набору данных, когда у нас были доступны   
все объекты, и попробуем получить от каждого объекта только значения поля NAME.   
Для этого вводим $[*].NAME (см. скрины ниже)

Так мы можем получить информацию по любым полям, меняя параметр NAME на любой другой. 

![](https://it-solution.kdb24.ru/api/v1/froala/file_proxy/370119/3165/739101/?sig=3c6fb3aa543df154454d02020be4546c)![](https://it-solution.kdb24.ru/api/v1/froala/file_proxy/370129/3165/739130/?sig=8a3cf86321ab21c6acea090630fecd03)

## Получить объекты по условию

Теперь, давайте попробуем получить объекты по условию. Условие следующее:   
найдем всех пользователей у которых значение поля NAME = Владислав.   
Для этого в JSONpath прописываем "$[*][?(@.NAME = "Владислав")"] (см. скрин ниже).

Таким образом, мы можем получить объекты по разным условиям, просто   
нужно заменить параметр, по которому будет вестись фильтрация (в данном случае параметр NAME)   
и заменить значение фильтра (в данном случае “Владислав”).

![](https://it-solution.kdb24.ru/api/v1/froala/file_proxy/370132/3165/739140/?sig=ed21b2bdead0ffdea5a58a97eff5ea6e)

## Узнать количество элементов списка

Если необходимо узнать количество возвращаемых элементов в списке, то можно использовать метод length.

Например, для получения количества задач можно использовать метод tasks.task.list и JSONPath: $.tasks[length()]

![](https://it-solution.kdb24.ru/api/v1/froala/file_proxy/543236/3165/1135956/?sig=2098deb48001fa19b7d1f3986d5d7e0a)
